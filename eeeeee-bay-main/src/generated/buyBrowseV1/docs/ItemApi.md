## checkCompatibility

> CompatibilityResponse checkCompatibility(itemId, contentType, opts)



This method checks if a product is compatible with the specified item. You can use this method to check the compatibility of cars, trucks, and motorcycles with a specific part listed on eBay.&lt;br&gt;&lt;br&gt;For example, to check the compatibility of a part, you pass in the &lt;b&gt;item_id&lt;/b&gt; of the part as a URI parameter and specify all the attributes used to define a specific car within the &lt;b&gt;compatibilityProperties&lt;/b&gt; container. If the call is successful, the response will be &lt;code&gt;COMPATIBLE&lt;/code&gt;, &lt;code&gt;NOT_COMPATIBLE&lt;/code&gt;, or &lt;code&gt;UNDETERMINED&lt;/code&gt;. Refer to &lt;a href&#x3D;\&quot;/api-docs/buy/browse/resources/item/methods/checkCompatibility#response.compatibilityStatus\&quot;&gt;compatibilityStatus&lt;/a&gt; for details.   &lt;br&gt;&lt;br&gt;&lt;span class&#x3D;\&quot;tablenote\&quot;&gt;&lt;b&gt;Note:&lt;/b&gt; The only products supported are cars, trucks, and motorcycles.&lt;/span&gt;&lt;br&gt;To find the attributes and values for a specific marketplace, you can use the compatibility methods in the &lt;a href&#x3D;\&quot;/api-docs/commerce/taxonomy/resources/methods\&quot; target&#x3D;\&quot;_blank\&quot;&gt;Taxonomy API&lt;/a&gt;. You can use this data to create menus to help buyers specify the product, such as their car.&lt;br&gt;&lt;br&gt;For more information and a list of required attributes for the US marketplace that describe motor vehicles, refer to &lt;a href&#x3D;\&quot;/api-docs/buy/static/api-browse.html#Check\&quot; target&#x3D;\&quot;_blank\&quot;&gt;Check compatibility&lt;/a&gt; in the &lt;i&gt;Buying Integration Guide&lt;/i&gt;.&lt;br&gt;&lt;br&gt;For an example, refer to the &lt;a href&#x3D;\&quot;/api-docs/buy/browse/resources/item/methods/checkCompatibility#h2-samples\&quot;&gt;Samples&lt;/a&gt; section.&lt;br&gt;&lt;br&gt;&lt;span class&#x3D;\&quot;tablenote\&quot;&gt;&lt;b&gt;Note:&lt;/b&gt; This method is supported in Sandbox but &lt;i&gt;only&lt;/i&gt; when passing in the specified &lt;code&gt;item_id&lt;/code&gt; and compatibility name-value pairs listed in &lt;a href&#x3D;\&quot;/api-docs/buy/browse/resources/item/methods/checkCompatibility#s0-1-22-6-7-7-6-SandboxSample-1\&quot;&gt;Sample 2: Sandbox Sample&lt;/a&gt;.&lt;/span&gt;&lt;h3&gt;Restrictions&lt;/h3&gt;For a list of supported sites and other restrictions, refer to &lt;a href&#x3D;\&quot;/api-docs/buy/browse/overview.html#API\&quot;&gt;API Restrictions&lt;/a&gt;.

## getItem

> Item getItem(itemId, opts)



This method retrieves the details of a specific item, such as description, price, category, all item aspects, condition, return policies, seller feedback and score, shipping options, shipping costs, estimated delivery, and other information the buyer needs to make a purchasing decision.&lt;br&gt;&lt;br&gt;The Buy APIs are designed to let you create an eBay shopping experience in your app or website. This means you will need to know when something, such as the availability, quantity, etc., has changed in any eBay item you are offering. This is easily achieved by setting the &lt;b&gt;fieldgroups&lt;/b&gt; URI parameter to one of the following values:&lt;ul&gt;&lt;li&gt;&lt;code&gt;COMPACT&lt;/code&gt;&lt;br&gt;Reduces the response to only those fields necessary in order to determine if any item detail has changed. This field group &lt;b&gt;must&lt;/b&gt; be used alone.&lt;/li&gt;&lt;li&gt;&lt;code&gt;PRODUCT&lt;/code&gt;&lt;br&gt;Adds fields to the default response that return information about the product/item. This field group may also be used in conjunction with &lt;code&gt;ADDITIONAL_SELLER_DETAILS&lt;/code&gt;.&lt;/li&gt;&lt;li&gt;&lt;code&gt;ADDITIONAL_SELLER_DETAILS&lt;/code&gt;&lt;br&gt;Adds an additional field to the response that returns the seller&#39;s user ID. This field group may also be used in conjunction with &lt;code&gt;PRODUCT&lt;/code&gt;.&lt;/li&gt;&lt;li&gt;&lt;code&gt;CHARITY_DETAILS&lt;/code&gt;&lt;br&gt;Adds additional fields to the response that return charity information associated with the item, if applicable.&lt;/li&gt;&lt;/ul&gt;For additional information, refer to &lt;a href&#x3D;\&quot;/api-docs/buy/browse/resources/item/methods/getItem#uri.fieldgroups\&quot;&gt;fieldgroups&lt;/a&gt;.&lt;h3&gt;Restrictions&lt;/h3&gt;For a list of supported sites and other restrictions, refer to &lt;a href&#x3D;\&quot;/api-docs/buy/browse/overview.html#API\&quot;&gt;API Restrictions&lt;/a&gt;.&lt;br&gt;&lt;br&gt;&lt;span class&#x3D;\&quot;tablenote\&quot;&gt;&lt;b&gt;eBay Partner Network:&lt;/b&gt; In order to be commissioned for your sales, you must use the URL returned in the &lt;code&gt;itemAffiliateWebUrl&lt;/code&gt; field to forward your buyer to the ebay.com site.&lt;/span&gt;

## getItemByLegacyId

> Item getItemByLegacyId(legacyItemId, opts)



This method is a bridge between the eBay legacy APIs, such as &lt;b&gt;Shopping&lt;/b&gt; and &lt;b&gt;Finding&lt;/b&gt;, and the eBay Buy APIs. There are differences between how legacy APIs and RESTful APIs return the identifier of an \&quot;item\&quot; and what the item ID represents. This method lets you use the legacy item ids retrieve the details of a specific item, such as description, price, and other information the buyer needs to make a purchasing decision. It also returns the RESTful &lt;b&gt;item_id&lt;/b&gt;, which you can use with all the Buy API  methods.&lt;br&gt;&lt;br&gt;For additional information about how to use legacy ids with the Buy APIs, refer to &lt;a href&#x3D;\&quot;/api-docs/buy/static/api-browse.html#Legacy\&quot; target&#x3D;\&quot;_blank\&quot;&gt;Item ID legacy API compatibility overview&lt;/a&gt; in the Buying Integration guide.&lt;/br&gt;&lt;br&gt;This method returns the item details and requires you to pass in either the &lt;b&gt;item_id&lt;/b&gt; of a non-variation item or the &lt;b&gt;item_id&lt;/b&gt; values for both the parent and child of an item group.&lt;br&gt;&lt;br&gt;&lt;span class&#x3D;\&quot;tablenote\&quot;&gt;&lt;b&gt;Note:&lt;/b&gt; An item group is an item that has various aspect differences, such as color, size, storage capacity, etc.&lt;/span&gt;&lt;/br&gt;When an item group is created, one of the item variations, such as the red shirt size L, is chosen as the \&quot;parent\&quot;. All other items in the group are the children, such as the blue shirt size L, red shirt size M, etc.&lt;br&gt;&lt;br&gt;The &lt;b&gt;fieldgroups&lt;/b&gt; URI parameter lets you control what is returned in the response:&lt;ul&gt;&lt;li&gt;Setting &lt;b&gt;fieldgroups&lt;/b&gt; to &lt;code&gt;PRODUCT&lt;/code&gt; adds additional fields to the default response that return information about the product of the item.&lt;/li&gt;&lt;li&gt;Setting the &lt;b&gt;fieldgroups&lt;/b&gt; to &lt;code&gt;ADDITIONAL_SELLER_DETAILS&lt;/code&gt; adds an additional field to the response that returns the seller&#39;s user ID.&lt;/li&gt;&lt;li&gt;Setting the &lt;b&gt;fieldgroups&lt;/b&gt; to &lt;code&gt;CHARITY_DETAILS&lt;/code&gt; adds additional fields to the response that return charity information associated with the item, if applicable.&lt;/li&gt;&lt;/ul&gt;These &lt;b&gt;fieldgroups&lt;/b&gt; can be used independently or at the same time. ## getItemsByItemGroup

> ItemGroup getItemsByItemGroup(itemGroupId, opts)



This method retrieves details about individual items in an item group. An item group is an item that has various aspect differences, such as color, size, storage capacity, etc.&lt;br&gt;&lt;br&gt;You pass in the &lt;b&gt;item_group_id&lt;/b&gt; as a URI parameter.&lt;br&gt;&lt;br&gt;This method returns two main containers:&lt;ul&gt;&lt;li&gt;&lt;b&gt;items&lt;/b&gt;&lt;br&gt;This container has an array of containers with the details about each item in the group.&lt;/li&gt;&lt;li&gt;&lt;b&gt;commonDescriptions&lt;/b&gt;&lt;br&gt;This container has an array of containers for a description and the &lt;code&gt;item_ids&lt;/code&gt; for all items that have this exact description. Because items within an item group often have the same description, this decreases the size of the response.&lt;/li&gt;&lt;/ul&gt;Setting the &lt;b&gt;fieldgroup&lt;/b&gt; to &lt;code&gt;ADDITIONAL_SELLER_DETAILS&lt;/code&gt; adds an additional field to the response that returns the seller&#39;s user ID. Setting the &lt;b&gt;fieldgroup&lt;/b&gt; to &lt;code&gt;CHARITY_DETAILS&lt;/code&gt; adds additional fields to the response that returns charity information associated with the item, if applicable. For more information, refer to &lt;a href&#x3D;\&quot;/api-docs/buy/browse/resources/item/methods/getItemsByItemGroup#uri.fieldgroups\&quot;&gt;fieldgroups&lt;/a&gt;.&lt;h3&gt;Restrictions&lt;/h3&gt;For a list of supported sites and other restrictions, refer to &lt;a href&#x3D;\&quot;/api-docs/buy/browse/overview.html#API\&quot;&gt;API Restrictions&lt;/a&gt;.&lt;br&gt;&lt;br&gt;&lt;span class&#x3D;\&quot;tablenote\&quot;&gt;&lt;b&gt;eBay Partner Network: &lt;/b&gt; In order to be commissioned for your sales, you must use the URL returned in the &lt;code&gt;itemAffiliateWebUrl&lt;/code&gt; field to forward your buyer to the ebay.com site.&lt;/span&gt;
